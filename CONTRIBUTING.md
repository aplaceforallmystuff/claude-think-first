# Contributing to Claude Think First

Thanks for your interest in improving the think-first skill! This skill helps enforce structured decision-making before implementation.

## Ways to Contribute

### Add Mental Models

If you know a decision-making framework not included:

1. Open an issue describing the model
2. Explain what types of decisions it's best for
3. Provide the key questions/prompts
4. Include examples of it in action

### Improve Model Prompts

The prompt questions for each model can always be refined:

- More specific questions
- Better sequencing
- Clearer language
- Additional context for when to use

### Enhance Detection

Help improve detection of decision moments:

- Additional trigger phrases
- Better categorization logic
- Edge cases that should invoke the skill

### Add Examples

Real-world examples showing the skill in action:

- Different decision types
- Various mental models applied
- Before/after comparisons

## Submitting Changes

1. Fork the repository
2. Create a feature branch (`git checkout -b add-new-model`)
3. Make your changes to `skills/think-first/SKILL.md`
4. Test with various decision scenarios
5. Submit a pull request

## Guidelines

### Keep It Practical

Mental models should be:
- Easy to remember
- Quick to apply (2-5 minutes)
- Genuinely useful for decisions

### Maintain Consistency

Follow the existing format:
- Model name in the table
- Key question summary
- Full prompt in `<model_prompts>` section
- Clear, numbered questions

### Model Selection Criteria

A good mental model for this skill:
- Applies to software/business decisions
- Can be worked through in a few minutes
- Produces actionable insights
- Has clear prompt questions

## Questions?

Open an issue for discussion before major changes.
